 总体介绍

AngularJs 1.x(之后就简称为ng了)的结构分为html模板和js代码。

html主模板只有一个， 然后由js驱动应用。(有2种驱动方式)

ng的主要作用是提供了一个抽取重复代码的逻辑，

代码的复用性强弱可以看出遵循ng设计的程度。

 ng的module种类： provider ,factory ,service ,value ,constant ,decorator ,animation ,filter ,controller ,directive ,component ,config ,run service

constant 配置环境变量，不可修改

value 全局变量，可以修改，不常用因为不能带逻辑

provider 全局变量；封装逻辑

provider

应用-封装底层服务($http, $log, $location)

应用-做队列、缓存/批量执行

结构-返回$get方法执行结果

结构-其余方法可在同名provider中调用，可用做工厂模式

 directive

语义化标签， 封装dom操作

分类：EMAC

误区：“不能用jquery”应该为不能使用id、class选择器

 组织代码 angular.module('app',['privatelibs','thirdpartlibs','env']) .config().run().factory().directive().controller().filter() promise

优点

避免callback hell

避免为了传播回调成功/错误而copy调用回调的代码

方便组织代码调用顺序逻辑

缺点

兼容问题：规范不统一

模式不兼容：有的时候建议使用事件监听

异常处理：每个链都需要手动捕获异常
